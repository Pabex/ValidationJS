Documentación ValidatorJS 1.1

Método createValidator( options: Json object ): Validator Object

Crea una instancia del validator asociada a un formulario (o elemento contenedor de campos a ser validados).

Parámetros:

  • options: Objeto JSON con las opciones para crear el validador.
Nombre Tipo Defecto Descripción
form jQuery object Obligatorio (sólo si no especificó un button)

Objeto jQuery que representa el formulario sobre el cual se realizarán las validaciones.

button jQuery object Obligatorio (sólo si no especificó un form)

Objeto jQuery que representa un botón que dispará las validaciones. Por ejemplo: se usaría en lugar de un form utilizo un div y en el mismo agrego un button que cause las validaciones, quizás para hacer una consulta AJAX si los campos son válidos.

triggers int[] []

Representa los eventos que dispararán la validación. Por defecto se puede aplicar de la siguiente forma:

  • Validar un formulario estándar: Sólo es necesario pasar como "form" el objeto jQuery que represente el formulario. Internamente se setea el evento VALIDATE_ON_FORM_SUBMIT.
  • Validar campos usando un botón: Es necesario pasar como "button" el objeto jQuery que represente el botón que dispará las validaciones. Internamente se setea el evento VALIDATE_ON_BUTTON_CLICK.

Los disparadores de validación se listan a continuación:

  • VALIDATE_ON_FORM_SUBMIT: El disparador es el evento submit (envío) del formulario dado por la opción "form". Cuando se envía el formulario y éste es inválido, automáticamente se evita el envío del mismo. Para usar este disparador es obligatorio que esté presente la opción "form".
  • VALIDATE_ON_BUTTON_CLICK: El disparador es el evento click del botón dado por la opción "button". Para usar este disparador es obligatorio que esté presente la opción "button".
  • VALIDATE_ON_FIELD_BLUR: El disparador es el evento "blur" (pérdida de foco) de los controles y campos del formulario.
  • VALIDATE_ON_FIELD_CHANGE: El disparador es el evento "change" (cambio de valor) de los controles y campos del formulario. Para los controles de tipo texto se dispara ante el evento "keyup" (cuando se suelta una tecla).
  • VALIDATE_ON_CUSTOM_EVENT:

    El disparador se da como una función genérica dada en la opción "validationCustomEvent" (obligatoria). La función posee la siguiente signatura:

    function validationCustomEvent(validateFunction) : void

    Donde validateFunction es la función que se debe se llamada dentro del evento personalizado como disparador de las validaciones, el mismo retrona "true" si las validaciones fueron correctas y "false" en caso de formulario inválido.

    Para usar este disparador es obligatorio que esté presente la opción "validationCustomEvent".

validationCustomEvent function undefined

Función que establece el evento disparador de las validaciones. Este evente puede ser cualquiera especificado por el usuario. Signatura:

function validationCustomEvent(validateFunction) : void

Donde los parámetros son:

  • validateFunction: Función que ejecuta las validaciones del formulario, retona si "true" si el formulario es válido o "false" si es inválido. Su signatura:

    function validate(evt: Event Object): boolean

    Donde los parámetros son:

    • evt: Objeto Event que disparó las validaciones
message string undefined Es el mensaje de error que se devuelve cuando el formulario es inválido.
validValidation function undefined

Función a ejecutarse cuando se verifica una validación de un campo del formulario como válida. Signatura:

function validValidation(options: { event: Event Object, validation: Validation Object, field: jQuery object }): void

Donde los parámetros son:

  • event: Objeto Event que disparó las validaciones.
  • validation: Objeto Validation que contiene toda la información de la validación ejecutada sobre el campo.
  • field: Objeto jQuery que representa el campo sobre el cual la validación se verificó como válida.
invalidValidation function undefined

Función a ejecutarse cuando se verifica una validación de un campo del formulario como inválida. Signatura:

function invalidValidation(options: { event: Event Object, validation: Validation Object, field: jQuery object, message: string }): void

Donde los parámetros son:

  • event: Objeto Event que disparó las validaciones.
  • validation: Objeto Validation que contiene toda la información de la validación ejecutada sobre el campo.
  • field: Objeto jQuery que representa el campo sobre el cual la validación se verificó como inválida.
  • message: Mensaje a mostrar informando la validación inválida.
validField function undefined

Función a ejecutarse cuando se verifica que todas las validaciones de un campo del formulario son válidas, por lo tanto el campo es válido. Signatura:

function validField(options: { event: Event Object, field: jQuery object }): void

Donde los parámetros son:

  • event: Objeto Event que disparó las validaciones.
  • field: Objeto jQuery que representa el campo sobre el cual las validaciones se verificaron como válidas.
invalidField function undefined

Función a ejecutarse cuando se verifica que al menos unas de las validaciones de un campo del formulario es inválido, por lo tanto el campo es inválido. Signatura:

function validField(options: { event: Event Object, field: jQuery object, messages: string[] }): void

Donde los parámetros son:

  • event: Objeto Event que disparó las validaciones.
  • field: Objeto jQuery que representa el campo inválido.
  • messages Arreglo de mensajes de todas aquellas validaciones que se verificaron como inválidas para el campo.
validForm function undefined

Función a ejecutarse cuando se verifica que el formulario es válido, cuando todas las validaciones de todos sos campos son verificadas como válidas. Signatura:

function validField(options: { event: Event Object, validator: Validator object, trigger: int[, form: jQuery object, button: jQuery object] }): void

Donde los parámetros son:

  • event: Objeto Event que disparó las validaciones.
  • validator: Objeto Validator que valida el formulario.
  • trigger: Número entero que representa el evento disparador de validaciones (VALIDATE_ON_FORM_SUBMIT, VALIDATE_ON_BUTTON_CLICK, VALIDATE_ON_FIELD_BLUR, VALIDATE_ON_FIELD_CHANGE o VALIDATE_ON_CUSTOM_EVENT).
  • form (opcional): Objeto jQuery que representa el formulario cuando el trigger (disparador de validaciones) es del tipo VALIDATE_ON_FORM_SUBMIT.
  • button (opcional): Objeto jQuery que representa el botón que dispara las validaciones cuando el trigger (disparador de validaciones) es del tipo VALIDATE_ON_BUTTON_CLICK.
invalidForm function undefined

Función a ejecutarse cuando se verifica que el formulario es inválido, el mismo es inválido cuando al menos una validación en uno de sus campos ser verifica como inválida. Signatura:

function validField(options: { event: Event Object, validator: Validator object, trigger: int[, form: jQuery object, button: jQuery objecy] }): void

Donde los parámetros son:

  • event: Objeto Event que disparó las validaciones.
  • validator: Objeto Validator que valida el formulario.
  • trigger: Número entero que representa el evento disparador de validaciones (VALIDATE_ON_FORM_SUBMIT, VALIDATE_ON_BUTTON_CLICK, VALIDATE_ON_FIELD_BLUR, VALIDATE_ON_FIELD_CHANGE o VALIDATE_ON_CUSTOM_EVENT).
  • form (opcional): Objeto jQuery que representa el formulario cuando el trigger (disparador de validaciones) es del tipo VALIDATE_ON_FORM_SUBMIT.
  • button (opcional): Objeto jQuery que representa el botón que dispara las validaciones cuando el trigger (disparador de validaciones) es del tipo VALIDATE_ON_BUTTON_CLICK.
before function undefined

Función a ejecutarse antes de realizar las validaciones del formulario. Signatura:

function before({ event: Event Object, fields: string[], trigger: int }): void

Donde los parámetros son:

  • event: Objeto Event que disparó las validaciones.
  • fields: Arreglo de los ids de los campos que conforman el formulario.
  • trigger: Trigger (id de disparador de evento) que disparó las validaciones. Puede ser null (por ejemplo si se usó como trigger un VALIDATE_ON_CUSTOM_EVENT).
after function undefined

Función a ejecutarse después de realizar las validaciones del formulario. Signatura:

function before({ event: Event Object, valid: boolean, trigger: int }): void

Donde los parámetros son:

  • event: Objeto Event que disparó las validaciones.
  • valid: Resultado de las validaciones: vale "true" cuando el formulario completo es válido, "false" caundo al menos una validación es inválida.
  • trigger: Trigger (id de disparador de evento) que disparó las validaciones. Puede ser null (por ejemplo si se usó como trigger un VALIDATE_ON_CUSTOM_EVENT).
Método validator.addValidation( field: jQuery object, validationType: int, options: Json object ): void

Agrega una validación específica para un campo del validator.

Parámetros:

  • field: Objeto jQuery que representa el campo al cual se va a agregar el nuevo tipo de validación. El objeto jQuery debe ser un selector por id únicamente.
  • validationType: Número entero que representa el tipo de validación.
  • options: Objeto JSON con las opciones para agregar la validación.
Nombre Tipo Defecto Descripción
field jQuery object Obligatorio

Objeto jQuery que representa el campo al cual se va a agregar el nuevo tipo de validación. El objeto jQuery debe ser un selector por id únicamente.

validationType int Obligatorio

Número entero que representa el tipo de validacón que se aplica al campo:

  • VALIDATION_TYPE_REQUIRED: Valida que se haya ingresado o seleccionado un valor en el campo.
  • VALIDATION_TYPE_LENGTH: Valida que el campo de texto posea la cantidad de caracteres impuesta como límite.
  • VALIDATION_TYPE_NUMBER: Valida que el campo posea un valor de tipo numérico. Puede ser entero o decimal, posito o negativo, en caso de ser decimal sólo puede usarse el punto como separador.
  • VALIDATION_TYPE_INT: Valida que el campo posea un valor numérico entero (positivo o negativo).
  • VALIDATION_TYPE_DECIMAL: Valida que el campo posea un valor numérico decimal (positivo o negativo). El caracter para separar los decimales es el punto.
  • VALIDATION_TYPE_VALUE: Valida que el valor del campo sea un número comprendido en los límites dados.
  • VALIDATION_TYPE_EMAIL: Valida que el campo posea una dirección de email válida.
  • VALIDATION_TYPE_GROUP: Valida que el grupo de elementos (sean checkboxes o radiobuttons) que posean al mismo grupo estén seleccionados con una cantidad que esté dentro de los límites dados.
  • VALIDATION_TYPE_RADIO_GROUP: Valida que el grupo de radiobuttons tenga un elemento seleccionado al menos.
  • VALIDATION_TYPE_CHECKBOX_GROUP: Valida que el grupo de checkboxes posea la cantidad de elementos seleccionados con una cantidad que esté dentro de los límites dados.
  • VALIDATION_TYPE_CUSTOM: Aplica una validación personalizada al campo usando una función cuyo parámetro es el valor del campo y retorna "true" si es válido y "false" en caso contrario.
options JSON undefined

Existen parámetros independientes del tipo de validación (validationType):

  • message: Es el mensaje que se mostrará en caso de verificar la validación como inválida.
  • required: Especifica que el campo es obligatorio (no debe ser vacío), este parámetro depende del tipo de validación.

También existen parámetros que son necesarios (algunos opcionales) para el funcionamiento de algunos tipos de validación:

  • VALIDATION_TYPE_REQUIRED: Valida que se haya ingresado o seleccionado un valor en el campo. Dependiendo del tipo de campo de control:
    • Campo de tipo texto (text, password, textarea): Valida que el valor del mismo tengas uno o más carateres distintos de blanco.
    • Campo de selección (select): Valida que el valor de la opción seleccionada no represente "nada seleccionado" dependiendo de los parámetros opcionales:
      • Sin ningún parámetro opcional (funcionamiento por defecto): Intenta convertir a número entero el valor de la opción seleccionada y valida que sea distinta de '-1' (menos uno).
      • Parámetro 'invalidRequiredValue': Es el valor de la opción que representa el vacío. El valor de la opción seleccionada se compara este parámetro y así validar si seleccionó algo el usuario.
      • Parámetro 'checkRequiredMethod': Representa la función de validación que determina cuando un 'select' tiene seleccionada una opción "vacío". Su signatura es:

        function checkRequiredMethod(value: string) : boolean

        Donde 'value' es el valor de la opción seleccionada y debería retornar "true" en caso de que la opción seleccionada represente vacío.

  • VALIDATION_TYPE_LENGTH: Valida que el campo de texto posea la cantidad de caracteres impuesta como límite. Parámetros opcionales:
    • Parámetro 'min': Representa la cantidad mínima de caracteres que deber tener el campo de texto.
    • Parámetro 'max': Representa la cantidad máxima de caracteres que deber tener el campo de texto.

    Observaciones: ambos parámetros son opcionales y no depende uno del otro. En el caso de que se establezca el parámtro 'max' pero no un mínimo, el campo puede ser vacío y sí es válido.

  • VALIDATION_TYPE_NUMBER: Valida que el campo posea un valor de tipo numérico. Puede ser entero o decimal, posito o negativo, en caso de ser decimal sólo puede usarse el punto como separador. No tiene parámetros opcionales.

    Observaciones: En caso de que el campo esté vacío se considera válido, porque no hay nada cargado. Esta validación valida que lo que esté cargado sea un número.

  • VALIDATION_TYPE_INT: Valida que el campo posea un valor numérico entero (positivo o negativo). Parámetros opcionales:
    • Parámetro 'min': Representa el valor entero mínimo aceptado.
    • Parámetro 'max': Representa el valor entero máximo aceptado.

    Observaciones: En caso de que el campo esté vacío se considera válido, porque no hay nada cargado. Esta validación valida que lo que esté cargado sea un número entero. Ambos parámetros son opcionales y no depende uno del otro.

  • VALIDATION_TYPE_DECIMAL: Valida que el campo posea un valor numérico decimal (positivo o negativo). El caracter para separar los decimales es el punto.
  • VALIDATION_TYPE_VALUE: Valida que el valor del campo sea un número comprendido en los límites dados. Parámetros obligatorios:
    • Parámetro 'min': Representa el mínimo número aceptado.
    • Parámetro 'max': Representa el máximo número aceptado.

    Observaciones: ambos parámetros son opcionales y no depende uno del otro.

  • VALIDATION_TYPE_EMAIL: Valida que el campo posea una dirección de email válida. El formato por defecto es el siguiente:

    [carácteres alfanuméricos]@[caracteres alfabéticos y el guión bajo].[de dos a tres carácteres alfabéticos]

    Parámmetros opcionales

    • Parámetro 'format': Representa la expresión regular a usar como reemplazo del formato dado por defecto.

    Observaciones: En caso de que el campo esté vacío la validación verifica el campo como válido, porque no hay caracter que contrastar.

  • VALIDATION_TYPE_GROUP: Valida que el grupo de elementos (sean checkboxes o radiobuttons) que provean al mismo grupo estén seleccionados con una cantidad que esté dentro de los límites dados. Documentación pendiente
  • VALIDATION_TYPE_RADIO_GROUP: Valida que el grupo de radiobuttons tenga un elemento seleccionado al menos. Documentación pendiente
  • VALIDATION_TYPE_CHECKBOX_GROUP: Valida que el grupo de checkboxes posea la cantidad de elementos seleccionados con una cantidad que esté dentro de los límites dados. Documentación pendiente
  • VALIDATION_TYPE_CUSTOM: Aplica una validación personalizada al campo usando una función cuyo parámetro es el valor del campo y retorna "true" si es válido y "false" en caso contrario. Parámetros obligatorios
    • Parámetro 'id': Valor identificatorio del nombre que se le dió a la validación personalizada con el método 'addCustomValidation'.

    Parámetros opcionales:

    • Parámetro 'parameters': Objeto JSON que representa los parámetros recibiría la función personalizada.

    Observaciones: Es necesario que la validación personalizada ya existe y esté asociada al validator al se agrega esta validación.

Método validator.removeValidation( field: jQuery object, validationType: int ): void

Elimina una validación específica para un campo del validator.

Parámetros:

  • field: Objeto jQuery que representa el campo donde se va a eliminar el tipo de validación. El objeto jQuery debe ser un selector por id únicamente.
  • validationType: Número entero que representa el tipo de validación.
Método addCustomValidation( id: string, method: function ): boolean

Agrega una validación personalizada al objeto global ValidatorJS, la misma podrá ser usada en cualquier instancia validator con el método 'addValidation' con validationType igual a 'VALIDATION_TYPE_CUSTOM'. Retorna "true" si la valición personalizada pude ser agregada, esto es cuando no existe otra validación personalizada con el mismo 'id'.

Parámetros:

  • id: Valor identificatorio del nombre de la validación personalizada. Debe ser único.
  • method: Función a ejecutarse al momento de tratar de validar el formulario y/o campo que tenga esa validación personalizada. Su signatura es la siguiente:

    function method ( value: string ): boolean

    Donde "value" es el valor del campo que posee en el momento de la validación, y debería retornar "true" en caso de cumpliar con la validación (es válido) y "false" en caso contrario.

Método getCustomValidation( id: string ): CustomValidation Object

Recupera una validación personalizada del objeto global ValidatorJS dado su "id". Retorna "null" en caso de encontrar la validación personalizada.

Parámetros:

  • id: Valor identificatorio del nombre de la validación personalizada a recuperar.
Método removeCustomValidation( id: string ): boolean

Elimina una validación personalizada del objeto global ValidatorJS dado su "id". Retorna "true" si pudo eliminar la validación personalizada, y "false" en caso contrario.

Parámetros:

  • id: Valor identificatorio del nombre de la validación personalizada a eliminar.
